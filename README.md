## 📋 Project Overview

A comprehensive data analytics capstone project analyzing the Indian mutual fund ecosystem using **Bluestock's** fund data. This project demonstrates end-to-end data engineering, exploratory analysis, and business intelligence capabilities.

### Key Statistics
- **32,778** transactions analyzed
- **₹3.5 billion** in transaction value processed
- **40** mutual funds tracked across **10 fund houses**
- **12 states** geographic coverage
- **4.5 years** of historical NAV data (2022-2026)
- **1.36** average Sharpe ratio (strong risk-adjusted returns)
- **14.09%** average 3-year fund returns

## 📊 Dataset Descriptions

### fund_master_clean.csv
Core fund characteristics and metadata

| Column | Type | Description |
|--------|------|-------------|
| amfi_code | int | Unique fund identifier (AMFI code) |
| fund_house | string | Fund house name (SBI, HDFC, ICICI, etc.) |
| scheme_name | string | Full fund scheme name |
| category | string | Fund category (Large Cap, Small Cap, Debt, etc.) |
| sub_category | string | More specific fund classification |
| plan | string | Investment plan (Regular, Direct, Growth) |
| launch_date | date | Fund launch date |
| benchmark | string | Fund's benchmark index |
| expense_ratio_pct | float | Annual expense ratio |
| exit_load_pct | float | Exit load percentage |
| min_sip_amount | int | Minimum SIP amount (₹) |
| min_lumpsum_amount | int | Minimum lump sum (₹) |
| fund_manager | string | Fund manager name |
| risk_category | string | Risk classification |

**Use Cases:**
- Fund filtering by category or fund house
- Cost analysis (expense ratios)
- Entry/exit requirement analysis

### performance_clean.csv
Performance metrics and rankings for all funds

| Column | Type | Description |
|--------|------|-------------|
| amfi_code | int | Unique fund identifier |
| return_1yr_pct | float | 1-year annualized return (%) |
| return_3yr_pct | float | 3-year annualized return (%) |
| return_5yr_pct | float | 5-year annualized return (%) |
| sharpe_ratio | float | Risk-adjusted return metric |
| sortino_ratio | float | Downside risk-adjusted return |
| beta | float | Market sensitivity |
| alpha | float | Excess return vs benchmark |
| std_dev_ann_pct | float | Annual volatility (%) |
| max_drawdown_pct | float | Maximum decline from peak (%) |
| aum_crore | int | Assets under management (₹ Cr) |
| rank_1yr_return | int | Ranking by 1-year return |
| rank_3yr_return | int | Ranking by 3-year return |
| rank_sharpe | int | Ranking by Sharpe ratio |

**Use Cases:**
- Performance comparison and ranking
- Risk assessment and portfolio construction
- Manager alpha evaluation

### transactions_clean.csv
Investor transaction records with demographics

| Column | Type | Description |
|--------|------|-------------|
| investor_id | string | Unique investor identifier |
| transaction_date | date | Date of transaction |
| amfi_code | int | Fund invested in |
| transaction_type | string | SIP / Lumpsum / Redemption |
| amount_inr | int | Transaction amount (₹) |
| state | string | Investor's state |
| city | string | Investor's city |
| city_tier | string | City tier classification |
| age_group | string | Investor age bracket |
| gender | string | Investor gender |
| annual_income_lakh | float | Annual income (₹ lakhs) |
| payment_mode | string | UPI / Cheque / Mandate / Net Banking |
| kyc_status | string | KYC verification status |

**Use Cases:**
- Investor behavior analysis
- Geographic market insights
- Demographic targeting
- Compliance monitoring

### nav_history_clean.csv
Daily Net Asset Value (NAV) tracking

| Column | Type | Description |
|--------|------|-------------|
| amfi_code | int | Fund identifier |
| date | date | NAV date |
| nav | float | Net Asset Value (₹) |

**Use Cases:**
- Return calculations
- Performance tracking over time
- Volatility analysis

### nav_performance_analysis.csv
Computed NAV returns from historical data

| Column | Type | Description |
|--------|------|-------------|
| amfi_code | int | Fund identifier |
| nav_return_pct | float | NAV growth percentage (2022-2026) |

**Use Cases:**
- Long-term performance validation
- Historical return verification

### Fund Performance
- Average **3-year return: 14.09%** (exceeds inflation)
- **Sharpe ratio: 1.36** (strong risk adjustment)
- Best performer: **SBI Small Cap Fund** at 23.39% 3-year return
- Top 5 funds show consistent outperformance

### Investor Demographics
- Primary demographic: **26-35 years (41.1%)**
- Average annual income: **₹26.2 lakhs**
- **92% KYC verification** (high compliance)
- **Balanced gender distribution** across investor base

### Geographic Distribution
- **Top market: Punjab** (2,965 transactions)
- Strong **Tier-2 market penetration**
- Multi-state presence indicates scalable reach
- Balanced regional investment pattern

### Market Dynamics
- **10 fund houses** with competitive presence
- **40 funds** across 12 categories

## 📈 Strategic Recommendations

### For Fund Houses
1. **Leverage SIP Growth:** 60% SIP penetration shows investor confidence; promote automated investing
2. **Target Tier-2 Markets:** Geographic analysis shows strong potential in non-metro cities
3. **Product Diversification:** Develop income-focused and balanced products for mid-income segments
4. **Digital First:** High UPI/Net Banking usage indicates tech-savvy investor base

### For Individual Investors
1. **SIP Discipline:** Consistent monthly investing delivers better long-term returns
2. **Fund Selection:** Prioritize funds with Sharpe ratio > 1.0
3. **Portfolio Balance:** Mix debt (30-40%), large cap (40-50%), small cap (10-20%)
4. **Long-Term Horizon:** 5+ year investment horizon recommended for equity funds

### For Regulators
1. **Maintain KYC Standards:** Current 92% verification rate sets good precedent
2. **Investor Education:** Expand financial literacy in Tier-2 markets
3. **Performance Transparency:** Standardize reporting for easier comparison
4. **Fraud Prevention:** Leverage demographic insights for risk profiling

## 🛠️ Project Architecture

### ETL Pipeline Flow
```
Input Sources
    ├── SQLite Database (5 tables)
    ├── CSV Files (3 files)
    └── External APIs (future)
         ↓
    [EXTRACTION]
    ├── Connection pooling
    ├── Data validation
    └── Schema verification
         ↓
    [TRANSFORMATION]
    ├── Date standardization
    ├── Type casting
    ├── Feature engineering
    └── Aggregations
         ↓
    [LOADING]
    ├── CSV exports
    ├── Quality checks
    └── Output validation
         ↓
Output Datasets (8 files)

    ├── Transaction data
```
